# 健康检查命令使用说明

## 快速开始

在任何项目中运行：
```bash
cd /path/to/your/project
claude

# 输入：
/health-check
```

## 检查内容

### 四大维度评估
1. **文档完整性** (0-10分)
   - README 是否完整
   - 功能映射是否存在
   - 架构决策是否记录

2. **测试覆盖度** (0-10分)
   - 核心功能是否有测试
   - 测试质量如何
   - 是否有难以测试的代码

3. **三者一致性** (0-10分)
   - 代码 vs 文档
   - 测试 vs 文档
   - 代码 vs 测试

4. **代码质量** (0-10分)
   - 文件/函数规模
   - 过度设计识别
   - 代码结构问题

**总分 40 分**

## 输出报告

会生成包含以下内容的完整报告：

```
📊 总体评分
🚨 关键问题（需要立即处理）
⚠️ 中等问题（建议尽快处理）
💡 改进建议（非紧急）
📋 详细发现
🎯 优先级行动计划
💪 长期改进建议
```

## 使用建议

### 首次使用
1. 在项目根目录运行 `/health-check`
2. 仔细阅读报告（重点看关键问题）
3. 根据优先级行动计划逐步改进

### 定期检查
- 活跃开发项目：每月1次
- 维护期项目：每季度1次
- 重构前后：各检查1次

### 评分参考

| 总分 | 状态 | 建议 |
|-----|------|------|
| 31-40 | 健康 | 保持现状，继续优化 |
| 25-30 | 一般 | 逐步改进，优先解决关键问题 |
| < 25 | 需要关注 | 优先解决基础问题，暂缓新功能 |

## 常见问题

**Q: 检查需要多长时间？**
A: 约 10-15 分钟（取决于项目规模）

**Q: 会修改我的代码吗？**
A: 不会，只做分析和生成报告

**Q: 如何保存报告？**
A: 报告会在对话中显示，可以复制保存到 `docs/health-check-[date].md`

**Q: 分数低怎么办？**
A: 不要慌，按照优先级行动计划逐步改进即可

## 核心原则

记住三位一体：
```
代码 (What & How)  ←→  测试 (Verify)
  ↑                      ↑
  └──────  文档 (Why) ──┘
```

三者必须保持一致，缺一不可！
