# 双文档体系生成

## 核心目标
为指定模块创建功能文档(README.md)和技术文档(TECHNICAL.md)，遵循职责分离原则。

## 文档职责分离

### README.md (功能导向)
- **面向**: 用户、测试、产品
- **内容**: 功能描述、使用流程、验证边界、用户场景
- **参考模板**: `commands/templates/docs/README_TEMPLATE.md`

### TECHNICAL.md (架构导向)
- **面向**: 开发者、架构师
- **内容**: 技术架构、组件职责、数据流转、扩展指南
- **参考模板**: `commands/templates/docs/TECHNICAL_TEMPLATE.md`

## 核心原则

### 职责分离
- README回答：做什么？怎么用？有什么价值？
- TECHNICAL回答：怎么实现？如何扩展？架构如何设计？
- 避免内容重复，互补协作

### 务实导向 ⭐⭐⭐
- **聚焦已实现功能**: 不列未来可能性，避免过度设计
- **真实代码示例**: 展示实际实现片段，不写伪代码模板
- **实际用户场景**: 描述真实操作流程，展示具体价值
- **真实组件名称**: 使用实际文件路径和组件名

### 质量标准
- 必须包含Mermaid架构图（至少2个）
- 组件职责表必须包含真实文件路径
- 数据流转图必须反映实际流程
- 两文档交叉引用明确

## 执行流程

1. **分析模块**: 识别入口文件、核心逻辑、数据流动、外部接口
2. **创建README**: 参考模板创建功能导向文档
3. **创建TECHNICAL**: 参考模板创建技术导向文档
4. **质量检查**: 参考模板中的检查清单验证
