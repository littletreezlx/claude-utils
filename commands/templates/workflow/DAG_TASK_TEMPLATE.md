## TASK ##
全局审视与收尾

**目标**：纵观所有阶段产出，整体梳理，将剩余工作和待完善内容写入项目根目录 TODO.md

**⚠️ 重要**：你没有前序任务的会话历史。必须通过以下方式了解已完成的工作：
- `git log --oneline -20` 和 `git diff --stat` 查看本轮变更
- 读取入口任务文件和 state.json 了解计划 vs 实际完成
- 扫描文件系统确认产出物是否存在

**执行步骤**：
1. 运行 `git log --oneline -20` 和 `git diff --stat HEAD~N` 了解本轮变更全貌
2. 读取入口任务文件，对比计划的 TASK 列表与实际产出
3. 评估整体完成度：哪些目标已达成？哪些有偏差？哪些未执行？
4. 自问：这个任务还剩下什么？还有什么可以进一步优化？
5. **直接写入项目根目录 TODO.md**（不依赖 /todo-write），包含：
   - 本次任务的背景和执行摘要
   - 已完成的关键变更清单（从 git log 提取）
   - 遗留问题、未完成项、待完善内容
   - 下一步可执行的行动项（每项需自包含：目标、核心文件、完成标志）

**完成标志**：
- [ ] TODO.md 已写入项目根目录且包含遗留事项和下一步行动
