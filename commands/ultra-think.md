# 深度分析

> 多维度深度思考，用于架构决策、复杂问题分析、技术选型

## 使用场景

```bash
/ultra-think 是否应该迁移到微服务架构？
/ultra-think 如何在降低成本的同时支撑 10 倍流量？
/ultra-think 新平台应该选择什么技术栈？
```

## 分析框架

### 1. 问题解构
- 提取核心挑战：$ARGUMENTS
- 识别利益相关者和约束条件
- 挖掘隐含需求和隐藏复杂性
- 质疑假设，暴露未知

### 2. 多维度分析

| 视角 | 关注点 |
|------|--------|
| **技术** | 可行性、可扩展性、性能、安全、技术债 |
| **业务** | ROI、上市时间、竞争优势、风险收益 |
| **用户** | 需求痛点、可用性、体验、边界场景 |
| **系统** | 全局影响、集成点、依赖耦合、涌现行为 |

### 3. 方案生成
- 至少 3-5 个不同方案
- 每个方案评估：优劣、复杂度、资源、风险、长期影响
- 包含常规方案和创新方案
- 考虑混合方案

### 4. 深度推演
- 详细实施计划
- 潜在陷阱和缓解策略
- 分阶段方案和 MVP
- 二阶、三阶效应分析
- 失败模式和恢复机制

### 5. 跨域思考
- 其他行业的类比
- 不同领域的设计模式
- 自然/生物系统的启发
- 创新组合

### 6. 挑战验证
- 魔鬼代言人视角
- 识别弱点和盲区
- "如果...会怎样" 场景
- 压力测试假设
- 意外后果分析

## 输出格式

```markdown
## 问题分析
- 核心挑战
- 关键约束
- 成功关键因素

## 方案选项
### 方案 1: [名称]
- 描述 / 优劣 / 实施路径 / 风险评估

### 方案 2: [名称]
...

## 推荐方案
- 推荐选择及理由
- 实施路线图
- 成功指标
- 风险缓解计划

## 补充视角
- 反向观点
- 未来考量
- 需进一步研究的领域
- 不确定性说明和置信度
```

## 思维原则

- **第一性原理**：回归基本事实
- **系统思维**：关注互联和反馈
- **概率思维**：处理不确定性
- **逆向思维**：思考应该避免什么
- **二阶思维**：考虑后果的后果
