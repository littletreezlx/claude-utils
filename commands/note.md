# 知识库速记

## 核心目标
将用户输入的内容（笔记、知识点、想法、摘录等）**零交互**地归档到知识库 `~/LittleTree_Projects/MyForest/`，自动判断新建或追加到已有文档。

## 输入格式
`/note <内容>`

内容可以是：
- 一段知识笔记（如"Kotlin 协程的取消机制是协作式的..."）
- 带主题标记的内容（如"#投资 美联储降息对..."）
- 一段想法或摘录
- 多条混合内容（自动拆分归档）

## 执行策略

### 1. 分类定位
根据内容语义，匹配知识库目录结构。**详细路由规则见 CLAUDE.md 第 6 节「目录导航语义（AI 路由配置）」**。

速查表：

| 目录 | 领域 |
|------|------|
| `01-技术/` | 编程、计算机科学、工程实践（通用知识） |
| `02-AI/` | 人工智能、提示词、AI 工具 |
| `03-工作/` | 特定工作项目的技术文档、汇报、求职 |
| `04-独立开发/` | 独立开发、产品、项目实战 |
| `05-学习/` | 学习方法、自我管理、心理成长、语言 |
| `06-投资/` | 宏观经济、交易、理财 |
| `07-数字生活/` | 软件工具、数码设备、家庭基础设施 |
| `99-系统/01-Inbox/` | **兜底**：无法明确分类时放这里 |

**跨领域判断**：按目标/场景归文件夹，按工具/手段打标签和双链。如果内容以 `#主题` 开头，优先按主题定位。

### 2. 目录层级规范

遵循 **「3层骨架 + 4层容忍 + 5层熔断」** 原则：

| 层级 | 规则 |
|------|------|
| L1-L3 | **骨架层**，直接使用已有目录 |
| L4 | **容忍层**，仅当该分类下 ≥5 篇文档时才建子目录；<5 篇用前缀命名法（如 `性能优化-APM.md`） |
| L5+ | **熔断**，禁止创建。用 Markdown 标题 + 大纲替代 |

**编号策略**：
- L1-L2 保留已有编号（`01-技术` → `99-系统`）
- L3 及以下**不编号**，按语义命名

### 3. 查重与定位
- 在目标目录下搜索是否已有相关主题的文档
- **有**：追加到已有文档的合适位置（保持结构一致）
- **无**：在目标目录下创建新文档
- 跨学科内容注重**关联**：在文档末尾用 `## 关联知识` 标注相关领域的文档链接（如行为经济学 → 心理学 + 投资）

### 4. 内容整理
将用户的原始输入整理为结构化知识：
- 提炼核心概念，补充必要上下文
- 使用清晰的标题层级组织
- 技术术语保留英文，解释用中文；财经/心理学术语用中文并备注英文（如 `认知失调 (Cognitive Dissonance)`）
- **保留用户原始信息的完整性**，整理但不删减关键内容
- 时政与财经内容保持**客观中立**，侧重事实与逻辑推演

### 5. 图示化思维
凡涉及复杂关系、流程或模型，**优先使用 Mermaid 图表**：

| 内容领域 | 推荐图表 |
|---------|---------|
| 技术/逻辑 | `graph TD` 流程图、`sequenceDiagram` 时序图 |
| 财经/时政 | `graph LR` 因果图、`quadrantChart` 四象限分析 |
| 心理/人文 | `mindmap` 思维导图、`graph TB` 层级图 |

### 6. 输出确认
完成后输出一行摘要：
```
✅ [新建/追加] <文件路径> — <简要说明>
```

## 约束条件
- **零交互**：不询问用户分类、文件名、格式等问题，全部自主决策
- **不破坏已有结构**：追加内容时保持原文档的格式风格
- **不创建根目录文件**：所有内容必须归入子目录
- **文件命名**：用中文或关键英文术语，简洁明了（如 `协程取消机制.md`、`React Server Components.md`）
- **禁止过度加工**：整理 ≠ 重写，保持用户原意
- **异物隔离**：知识库内只存知识文档，不放代码/SDK/二进制文件；如需引用外部文件用 `file:///` 链接
- **新建目录前检查**：① 层级 ≤ 4？ ② 预期文档数 ≥ 5？ ③ 与已有目录无语义重叠？
- **禁止运行代码**：所有代码块仅作文本展示
