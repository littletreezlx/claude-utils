# 知识库速记

## 核心目标
将用户输入**零交互**地归档到知识库，自动判断新建或追加到已有文档。

## 输入格式
`/note <内容>`

内容可以是：
- 一段知识笔记（如"Kotlin 协程的取消机制是协作式的..."）
- 带主题标记的内容（如"#投资 美联储降息对..."）
- 一段想法或摘录
- 多条混合内容（自动拆分归档）

## 执行策略

### 1. 分类定位
路由规则遵循 `CLAUDE.md` 第 6 节「目录导航语义」。

### 2. /note 特有逻辑
- **多内容自动拆分**：多条混合内容按主题拆分归档
- **`#主题` 优先**：输入以 `#主题` 开头时，按主题定位优先
- **追加优先**：先查重，有相关文档则追加

### 3. 内容整理
遵循 `CLAUDE.md` 5.1 文档模板规范：
- 提炼核心概念，补充必要上下文
- 技术术语保留英文，解释用中文
- 保留用户原始信息的完整性
- 复杂关系优先使用 Mermaid 图表

### 4. 图示化
遵循 `CLAUDE.md` 第 2 节 Mermaid 要求。

## 约束条件
- **零交互**：不询问用户分类、文件名、格式等问题
- **不破坏已有结构**：追加内容时保持原文档格式风格
- **保留原始完整性**：整理 ≠ 重写，保持用户原意
- **禁止运行代码**：所有代码块仅作文本展示

## 输出确认
完成后输出一行摘要：
```
✅ [新建/追加] <文件路径> — <简要说明>
```
