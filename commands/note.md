# 知识库速记

## 核心目标
将用户输入的内容（笔记、知识点、想法、摘录等）**零交互**地归档到知识库 `/Users/zhanglingxiao/SynologyDrive/MyForest/`，自动判断新建或追加到已有文档。

## 输入格式
`/kb <内容>`

内容可以是：
- 一段知识笔记（如"Kotlin 协程的取消机制是协作式的..."）
- 带主题标记的内容（如"#投资 美联储降息对..."）
- 一段想法或摘录
- 多条混合内容（自动拆分归档）

## 执行策略

### 1. 分类定位
根据内容语义，匹配知识库目录结构：

| 目录 | 领域 |
|------|------|
| `01-技术/` | 编程、计算机科学、工程实践 |
| `02-AI/` | 人工智能、提示词、AI 工具 |
| `03-工作/` | 工作相关文档 |
| `04-独立开发/` | 独立开发、产品、项目 |
| `05-学习/` | 学习方法、自我管理、心理成长、语言 |
| `06-投资/` | 宏观经济、交易、理财 |
| `07-数字生活/` | 软件工具、数码设备 |
| `99-系统/01-Inbox/` | **兜底**：无法明确分类时放这里 |

如果内容以 `#主题` 开头，优先按主题定位。

### 2. 查重与定位
- 在目标目录下搜索是否已有相关主题的文档
- **有**：追加到已有文档的合适位置（保持结构一致）
- **无**：在目标目录下创建新文档

### 3. 内容整理
将用户的原始输入整理为结构化知识：
- 提炼核心概念，补充必要上下文
- 使用清晰的标题层级组织
- 技术术语保留英文，解释用中文
- 复杂关系用 Mermaid 图表可视化（遵循知识库 CLAUDE.md 的图示化原则）
- **保留用户原始信息的完整性**，整理但不删减关键内容

### 4. 输出确认
完成后输出一行摘要：
```
✅ [新建/追加] <文件路径> — <简要说明>
```

## 约束条件
- **零交互**：不询问用户分类、文件名、格式等问题，全部自主决策
- **不破坏已有结构**：追加内容时保持原文档的格式风格
- **不创建根目录文件**：所有内容必须归入子目录
- **文件命名**：用中文或关键英文术语，简洁明了（如 `协程取消机制.md`、`React Server Components.md`）
- **禁止过度加工**：整理 ≠ 重写，保持用户原意
