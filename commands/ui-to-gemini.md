# UI 截图转 Gemini 素材

## 目标
将客户端页面截图转换为结构化的产品功能与 UI 描述，作为交付给 Gemini 进行 UI 优化的基础素材。

## 前置条件 ⚠️
**必须提供页面截图**。

- ✅ 有截图 → 继续分析
- ❌ 无截图 → 终止并提示：「请提供页面截图。可使用 `/screen` 命令获取，或直接上传图片。」

## 执行方式

### 截图深度分析

从截图中提取以下信息：

**功能层**
- 页面类型与定位
- 解决的用户问题
- 核心业务流程
- 数据与状态

**UI 层**
- 布局结构
- 组件构成
- 视觉层次
- 交互区域

### 生成描述文档

输出格式（直接可复制给 Gemini）：

```markdown
# [页面名称] - UI 描述

## 页面概述
- **类型**：{列表页/详情页/表单页/设置页/对话页...}
- **核心功能**：{一句话描述这个页面要解决什么问题}

## 功能逻辑

### 用户场景
{用户在什么情况下会来到这个页面，要完成什么任务}

### 业务流程
{页面承载的业务逻辑，用户操作的流程}

### 数据说明
{页面展示了哪些数据，数据之间的关系}

## UI 结构

### 布局
{整体布局描述：顶部/内容区/底部等}

### 组件清单
| 组件 | 位置 | 功能 |
|------|------|------|
| ... | ... | ... |

### 视觉层次
- **主要信息**：{用户第一眼应该看到的}
- **次要信息**：{支撑主要信息的内容}
- **辅助信息**：{补充说明}

### 交互说明
{可点击/滑动/输入的区域及其预期行为}
```

## 输出要求

- 客观描述现状，不做优化建议
- 功能逻辑完整，Gemini 无需看代码即可理解
- UI 描述具体到组件级别
- 输出后提醒：「请将此文档和截图一起发给 Gemini，并补充你的优化需求。」

## 执行参数
`$ARGUMENTS` - 可选的补充说明（如页面名称、业务背景）
