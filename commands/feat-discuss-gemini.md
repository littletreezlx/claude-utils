# Gemini 产品方案咨询流

## 目标
利用 Claude Code 读取本地代码的优势，为 Gemini 准备一份高密度的上下文 Prompt，以获取高质量的产品设计或交互逻辑方案。

## 执行流程

### Step 1: 意图与上下文分析
1.  **理解需求**: 用户想做什么？(e.g., "设计会员中心" 或 "优化结账流程")
2.  **分类任务**:
    - **TYPE A (逻辑/交互)**: 涉及流程、状态、数据流转。
    - **TYPE B (视觉/风格)**: 涉及布局、配色、动效。
3.  **提取上下文 (Crucial)**:
    - 扫描 `routes` / `pages`: 当前功能入口在哪里？
    - 扫描 `schema` / `models`: 涉及哪些数据实体？
    - 扫描 `theme` / `ui-lib`: 现有的 UI 限制是什么？

### Step 2: 生成 Gemini 专用 Prompt
不要直接回答问题，而是生成一个被 `------- GEMINI BRIDGE START -------` 包裹的 Prompt 块。

请根据 **TYPE A** 或 **TYPE B** 选择对应的模板填充。

#### 模板 A: 针对逻辑/交互 (Product Logic & Flow)
如果用户关注功能流程：
- **Role**: Senior Product Designer (专注于 UX, User Flow, Edge Cases)
- **Context Section**: 填入提取到的 Sitemap 和 Data Models。
- **Task**: 要求 Gemini 输出 **User Journey**, **State Definition (Empty/Loading/Error)**, 和 **Logic Rules**。

#### 模板 B: 针对视觉/界面 (Visual & UI)
如果用户关注好看、布局：
- **Role**: Creative Director (专注于 Aesthetics, Layout, Motion)
- **Context Section**: 填入提取到的 Theme Colors, Font, 和 UI Lib (e.g. Tailwind/Flutter)。
- **Task**: 要求 Gemini 输出 **Component Anatomy**, **Layout Topology**, 和 **Animation Specs**。

### Step 3: 输出格式

请直接输出以下内容供用户复制：

```text

  [Claude 提取的路由/页面结构]
- **Data Model**:

- **Tech Constraints**: 
  [Claude 提取的技术栈，如 Flutter/React, Material/Shadcn]

## User Request
[用户原始输入]

## Instructions
请基于上述上下文，提供专业的解决方案。
