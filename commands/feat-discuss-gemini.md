# Gemini UI 方案咨询

## 目标
收集项目上下文，生成提示词供用户与 Gemini 交流，获取**产品设计方案**（非代码实现）。

## 执行方式

### 第一阶段：需求理解
1. **聚焦核心需求** — 1-2 个问题
2. **收集上下文**：数据模型、现有界面、技术栈
3. **绘制 ASCII 草图**（可选）— 帮助理解现状

### 第二阶段：生成提示词

````
------- GEMINI PROMPT START -------

# [功能名称] 产品方案咨询

## 背景
[项目类型、目标用户、核心场景]

## 当前状态
[现有界面 / ASCII 草图]

## 需求
[用户想要什么]

## 业务数据
```
[相关数据结构，帮助理解业务实体和关系]
```

## 技术约束
- 平台：[Flutter/React/等]
- 限制：[如有特殊限制]

## 期望输出

**请提供方案描述，不需要代码。**

1. **用户旅程**：核心场景下用户如何完成任务
2. **信息架构**：页面/模块如何组织，信息层级
3. **交互流程**：关键操作的步骤和反馈
4. **界面草图**：用 ASCII 图或结构化描述说明布局
5. **设计决策**：为什么这样设计（权衡取舍）

可选补充：
- 边界情况处理思路（空态、错误、加载）
- 可扩展性考虑

------- GEMINI PROMPT END -------
````

### 第三阶段：衔接说明

```
请将提示词发送给 Gemini，获取方案后粘贴给我。
我会基于方案进行技术实现。
```

## 约束条件
- **快速**：理解阶段 ≤ 2 轮对话
- **聚焦**：收集必要上下文，不堆砌
- **可用**：提示词可直接复制

## 后续衔接
- 简单功能 → 直接实现
- 复杂功能 → `/feat-discuss` 细化
- 大型功能 → `/todo-huge-task` 拆分
