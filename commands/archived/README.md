# 归档文件夹说明

> 本目录用于保存有价值但不再活跃使用的历史文档和脚本

## 📋 归档原则

### 应该归档的文件
- ✅ **已完成项目的总结文档** - 设计决策、实现笔记、经验总结
- ✅ **历史版本的辅助脚本** - 已被更好方案替代但有参考价值的脚本
- ✅ **重要的开发记录** - 架构演进、重构过程、技术调研
- ✅ **测试案例和实验** - 成功的测试方法、失败的尝试和教训

### 不应该归档的文件
- ❌ **活跃使用的命令文件** - 应该保留在 commands/ 根目录
- ❌ **模板和规范** - 应该在 templates/ 目录维护
- ❌ **临时测试文件** - 无参考价值，直接删除
- ❌ **通用工具脚本** - 不属于项目特定内容，直接删除

## 📁 目录结构

```
archived/
├── README.md                    # 本说明文件
├── decisions/                   # 历史架构决策记录
│   └── DAG_SYSTEM_DESIGN.md    # DAG 系统设计总结（示例）
├── experiments/                 # 实验和测试记录
│   └── test-results/           # 测试结果归档
└── scripts/                     # 历史辅助脚本
    └── legacy-helpers/         # 已废弃的辅助脚本
```

## 🗂️ 归档时机

### 项目完成时
- 将项目总结文档移入 `decisions/`
- 保留实现过程中的重要决策记录
- 归档测试结果和验证数据

### 重构替换时
- 旧版本脚本移入 `scripts/legacy-*`
- 保留设计演进的关键节点
- 记录替换原因和新方案优势

### 定期清理时
- 每季度检查 commands/ 根目录
- 识别不再活跃的文档和脚本
- 评估价值后决定归档或删除
