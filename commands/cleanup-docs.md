# 文档清理和归档

> **用途**：扫描项目根目录临时文件，评估价值，提供归档或清理方案

---

## 🚀 执行流程

### Step 1: 扫描临时文件

**识别模式**：
- 日期命名：`*-2024-*.md`, `*-20241104.md`
- 临时名称：`summary.md`, `notes.md`, `todo.md`, `temp-*.md`, `draft-*.md`
- Debug 相关：`debug-*.md`, `log-*.md`, `bugfix-*.md`

**排除**：README.md, CHANGELOG.md, CLAUDE.md, `docs/`, `.github/`

---

### Step 2: 内容分析和价值评估

**内容分类**：
- 架构决策 | 功能说明 | 重构记录 | API 变更 | 故障排查 | 开发日志 | 临时笔记

**价值评分**：
| 等级 | 标准 | 行动 |
|------|------|------|
| ⭐⭐⭐ 高 | 架构决策、复杂问题方案、功能实现说明 | 必须保留 |
| ⭐⭐ 中 | 常见问题方案、API 说明、配置说明 | 建议保留 |
| ⭐ 低 | 已体现在代码中、过时信息、简单 bug 记录 | 可以删除 |
| ❌ 无 | 完全过时、重复、空文件 | 应该删除 |

**归档位置**：
| 内容类型 | 推荐位置 |
|---------|---------|
| 架构决策 | `docs/architecture/` |
| 功能说明 | `docs/features/` |
| API 文档 | `docs/api/` |
| 故障排查 | `docs/troubleshooting/` |

---

### Step 3: 生成整理方案

```markdown
## 文档整理方案

### 📊 扫描结果
- 发现临时文件：X 个
- 高/中/低价值：X/X/X 个
- 建议删除：X 个

### 🎯 整理行动计划

#### 高优先级
1. **`文件名`** → 归档到 `docs/xxx/`
   - 价值：⭐⭐⭐ | 类型：架构决策
   - 行动：`mv 文件名 docs/xxx/`

#### 低优先级（删除）
1. **`文件名`** → 删除
   - 原因：内容过时
   - 行动：`rm 文件名`
```

---

## 📋 整理原则

1. **价值优先** - 高价值必须保留，低价值果断删除
2. **分类归档** - 按内容类型归档到对应目录
3. **合并整合** - 优先合并到现有文档
4. **提炼精华** - 归档时移除临时内容

---

## ⚠️ 注意事项

- 不要删除未分析的文件
- 归档前先创建目录
- 建议先 `git commit` 当前状态
- 不确定时保留并标注待处理
