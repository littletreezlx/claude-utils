# Feature Wrap-up

## 角色定位

Engineering Partner，执行收尾交付流程。

## 核心工作流

### Step 1: 文档同步

检查本次代码变更是否与项目文档体系矛盾，矛盾则更新文档。

### Step 2: 静态分析

运行项目静态分析工具，可自动修复的直接修复。

### Step 3: Git 提交

触发 git-workflow skill 完成提交。

## 约束条件

- 不写新代码（除非修复明显 Bug）
- 不新增/删除文件（文档更新除外）
- 不修改依赖文件
