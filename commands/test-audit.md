# 测试基础设施审计与修复

> 审计测试基础设施（能不能跑），直接修复缺失的配置和脚本

```bash
/test-audit                    # 审计并修复当前项目
/test-audit $ARGUMENTS         # 指定关注点（unit/integration/e2e）
```

## 核心原则

1. **审计即修复** — 发现问题直接解决，不询问
2. **只动基础设施** — 不修改业务代码和测试用例
3. **技术栈适配** — 根据项目类型选择方案

---

## 检查项

- [ ] 测试命令配置完整（unit/integration/e2e/watch/coverage）
- [ ] E2E 脚本具备：进程清理、依赖服务管理、报告生成
- [ ] 测试可正常运行（执行验证，不只是检查配置）
- [ ] CI/CD 兼容（环境变量、CI=true 行为差异、时区/Locale）

**缺失时**：根据项目实际情况直接生成适配的脚本和配置。

## 完成后

- 在对话中输出审计结果（项目类型、框架、各类型测试状态、已修复项）
- 建议更新 `CLAUDE.md` 的测试命令部分（列出建议内容，由用户确认）

## 禁止事项

1. 不使用模板文件 — 根据项目实际情况生成
2. 不生成报告文件 — 只在对话中输出
3. 不修改业务代码 — 只处理测试基础设施
