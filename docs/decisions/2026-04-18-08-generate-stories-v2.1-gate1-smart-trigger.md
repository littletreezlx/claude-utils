# 2026-04-18-08 generate-stories v2.1：Gate 1 从必经改为条件触发

## 背景

v2.0 落地当日（2026-04-18），Founder 立即在 `flametree_pick` 项目试跑 `/generate-stories`。Phase 1/2 按预期执行（事实摘要 201 行 / 覆盖率 95%），但 Gate 1 的 UX 暴露问题：

Founder 反馈：
> "我感觉是不是可以更加自动化一些，这也要我给意见吗？我感觉 AI 可以自己处理啊。"

## 诊断

v2.0 的 Gate 1 把**两种不同性质的决策**混在一起，全甩给 Founder：

| 决策内容 | 谁能判断 | v2.0 现状 |
|---------|---------|----------|
| 覆盖率是否达标（10 条覆盖 78/82 = 95%）| 🤖 AI（客观指标）| ❌ 塞进 Gate |
| 摇一摇是否独立 Story（SOUL 已说是核心差异化）| 🤖 AI（读 SOUL 可自主判断）| ❌ 塞进 Gate |
| 异常路径是否合并（按"≤8 步"约束自查）| 🤖 AI（结构规则明确）| ❌ 塞进 Gate |
| 端点缺失是否保留占位 Story（已有 `# TODO` 机制）| 🤖 AI（规则明确）| ❌ 塞进 Gate |
| 真正产品优先级分叉（规则未覆盖的边界判断）| 👤 Founder | ✅ 应 Gate |

**违反的原则**：
- CLAUDE.md 铁律「AI-Only 开发，AI 全权负责代码/测试/文档」
- 全局指引「Founder 时间预算 <5 分钟」
- v2.0 自己的设计目标：Gate 应该是"少数边界情况的人机接口"，不是"全对账表审核"

**根因**：v2.0 写 Gate 1 时过度保守，把"让 Founder 审一眼"当兜底，结果等于什么都不让 AI 决策。

## 决策

**v2.0 → v2.1，Gate 1 从"必经"改为"条件触发"。Gate 2 保留必经。**

### Gate 1 条件触发规则

AI 在 Phase 2 末尾做自查，**全部满足则自动通过进入 Phase 3**：

1. **覆盖率 ≥ 85%**（Behavior Audit 算出）
2. **落选行为 100% 有明确理由**（不能出现"原因未知"）
3. **端点缺失 ≤ 30%**（`/providers` 对账）
4. **无产品优先级分叉**（见下方定义）

自动通过时只打印一行报告，不展示全表：
```
✅ Gate 1 自动通过：覆盖率 95% (78/82)，端点缺失 8%，无产品优先级分叉。进入 Phase 3。
```

### Gate 触发条件（任一命中）

只要以下任一条件命中，Gate 弹出，但**只展示相关分叉，不展示全对账表**：

1. 覆盖率 < 85% 且剩余行为无法归因 → 问 Founder 是否补
2. 端点缺失 > 30% → 展示缺失端点清单，问是否继续生成（占位 Story 的意义存疑）
3. **产品优先级分叉**（AI 读完 SOUL/ROADMAP/features 仍无法决定）→ 展示该分叉选项，问 Founder

### "产品优先级分叉"的定义

| 场景 | 是否分叉 | AI 怎么判断 |
|------|---------|-----------|
| ROADMAP 明确"分享是 v2 计划" | ❌ 非分叉 | 明确：现在不写占位 |
| SOUL 明确"AI 语音是核心差异化" | ❌ 非分叉 | 明确：必写 Story |
| 两候选 Story 合并会超 8 步 | ❌ 非分叉 | 按"≤8 步"约束拆分 |
| 两功能同等重要、拆分还是合并无规则指引 | ✅ 分叉 | 问 Founder |
| 某功能 features/ 未写明优先级 + ROADMAP 未提 | ✅ 分叉 | 问 Founder |

### Gate 2 保持必经

不变。产品业务准确性是 AI 决策链的终点兜底，必须人审最终产物。

## 不改的事情

- DAG 流水线结构（5 Phase）不变
- 全覆盖模式不变
- Phase 3/4 的独立上下文规则不变
- 断言严格性标准不变

## 风险与反面检验

- **AI 自查不严可能跳过真正的分叉**：缓解方式——把"自动通过"的打印做得显式（列出覆盖率 / 落选数 / 端点缺失率），Founder 看到数据可以打断。
- **85% / 30% 阈值是拍脑袋**：首次运行数据出来后校准。
- **"产品优先级分叉"的定义依赖 AI 判断能力**：如果 AI 漏判，Gate 2 会兜底（业务不准确会被 Founder 打回）。

## 回滚条件

3 个月内观察到：
- Gate 1 自动通过率 < 50% → 阈值设得太严，放宽
- Founder 在 Gate 2 频繁发现"应该在 Gate 1 问我的东西"→ "产品优先级分叉"的规则不足，补定义
- 两次以上出现 Gate 1 自动通过但 Gate 2 全盘驳回 → 整体架构有问题，重新评估

## 触发背景

同日 2026-04-18 的改动链：
- 早些时候：generate-stories v1.0 → v2.0（decision 07，DAG 全覆盖模式）
- 稍后：Founder 实操反馈 → 本次 v2.0 → v2.1

v2.1 是 v2.0 的**实操校准**，不是架构变更。属于 AI-Only 协作迭代的正常节奏。
