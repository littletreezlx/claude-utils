# 2026-04-18-04 /think 升级模板必填 `[Claude Code 综合推荐]` 字段

**类型**：Skill 契约紧缩
**影响半径**：`~/.claude/skills/think/SKILL.md`
**版本**：think 0.5.1 → 0.5.2

## 问题

0.5.1 升级模板要求列 ≤4 个互斥选项 + 反面检验 + 决策勾选框，但**没要求 Claude Code 明确推荐哪个选项**。Founder 读完要自己做三方综合判断（事前验尸 + Gemini + GPT-5.4），这正是 `/think` 应该替他做的部分。

Founder 原话："虽然给了我一些选项，但是我还是希望 Claude Code 最后给我一个建议的总结方案，大部分情况下我会直接按照建议来。因为这是 3 个顶级 AI 的思考结果。"

AI 已经沉淀了三方视角、做完了判优、识别了触发条件，却在最后一步"哪个选项落地"把决策负担丢回 Founder——这是工作没做完。

## 决定

升级模板新增**必填字段** `[Claude Code 综合推荐]`，位置在"反面检验"之后、"决策选项"之前：

```markdown
- **Claude Code 综合推荐**: 采纳 <A|B|C|D> —— [≤100 字：三方（事前验尸 + Gemini + GPT-5.4）整合后我为什么推荐这个；选它你会得到什么、失去什么；为什么不是最接近的替代 Y]
```

**强制要求（校验条件）：**

1. **必须指向一个具体选项**（A/B/C/D 之一）。禁止回避措辞："看情况"、"看 Founder 偏好"、"视实测结果"等——三方 AI 判断已沉淀，Claude Code 作为 synthesizer 必须给出单一推荐
2. **必须包含反面对比**：不只说"选 X 的理由"，还要说"为什么不是最接近的替代 Y"——这是对 Founder 决策权的尊重（看到权衡而非单向推销）
3. **不允许"组合推荐"**：互斥性约束已保证选项不重叠，推荐就是挑一个
4. **≤100 字**：与核心主张字段长度对齐

## 为什么不选其他方案

### X1：把推荐做成可选字段

**否决原因**：可选即等于 Claude Code 会在"想不清楚"时偷懒省略，Founder 还是要自己综合。Founder 明确要求默认情况他直接按建议执行，意味着推荐是**决策闭环的主干**，不是装饰。

### X2：允许推荐"不下结论"（比如"需要更多信息"）

**否决原因**：
- 如果真的信息不足，应该在 Digest `[我的立场]` 里填"反驳"触发 Turn 2，而不是走到升级模板还摆烂
- 走到升级模板意味着已经触发 3.3.1 升级条件（决策影响 ≥1 天 / 持久化规则 / 产品灵魂 / 重型依赖）——这些影响面的事必须有推荐，拖着不推荐是管理层的失职

### X3：推荐由 Gemini 或 GPT-5.4 给（让外部 AI 决断）

**否决原因**：
- 外部 AI 不知道项目完整上下文（PRODUCT_SOUL / ROADMAP / TODO 等）——Claude Code 当轮会话里持有全部
- 两路外部 AI 经常意见分裂（本次 flametree_ai 实测就相反），谁的推荐算数？判优本身已经是 Claude Code 做的事，推荐自然由 Claude Code 完成
- "3 个顶级 AI" 指的就是 Gemini + GPT + Claude Code 三方，Claude Code 就是最后的综合节点

### X4：多个推荐（"推荐 A 为主 + 同时做 B 一部分"）

**否决原因**：
- 违反互斥性约束（0.5.1）——选项设计时就该保证两两互斥，真需要组合就合并为新选项 C
- 多个推荐等于没推荐，Founder 还是要自己在 A/B 之间再选一次
