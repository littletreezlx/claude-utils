# 2026-04-18-03 /think skill 0.5.1 — 首次真实运行后的契约缝隙修补

**类型**：Skill 契约紧缩（基于生产观察）
**影响半径**：`~/.claude/skills/think/SKILL.md`
**版本**：think 0.5.0 → 0.5.1

## 触发场景

2026-04-18 flametree_ai 项目 `/think 当前项目音频对话角色这套音频系统的架构和交互是否合理？` 是 0.5.0 DUAL 模式上线后的首次真实运行。产出通过了 7 字段 Digest + 升级模板结构性校验，但 Founder 评审后发现 4 个语义级缝隙，0.5.0 契约没写清或定得过严。

## 暴露的问题

### 问题 1：核心主张字段膨胀

Skill 0.5.0 规定 `[Gemini 核心主张]` / `[GPT-5.4 核心主张]` "一句话结论；禁止列事项清单"。首次运行实际输出里两个字段都膨胀到 3-4 句并塞入战术细节（"端侧加最简 RMS 能量 VAD Gate + 拆除 voiceProfile JSON blob + voice session 旁路化 + 纠正流量伪命题…"），核心主张变成小型决策清单。

**Founder 反馈**："一句话≤40 字" 之类硬量化约束太强，但 "一句话" 无约束等于 AI 想写多长就多长。

### 问题 2：Digest 和升级模板在升级路径下出现双重展开

升级路径下，Digest 已有 `[Gemini 核心主张]` `[GPT-5.4 核心主张]` `[模型判优]` `[我的立场]`，升级模板又重复 Gemini/GPT 核心主张 + 模型判优 + Claude Code 立场。Founder 读到两次同样的概括。Skill 0.5.0 没明确两者分工。

### 问题 3：决策选项数量失控

0.5.0 升级模板示意为 A/B + 再议/驳回。首次运行实际扩展到 **A/B/C/D + 组合 B+部分 A + 再议 + 驳回 = 6 个路径**。其中 3 个是嵌套关系（A 是全套、C 是 A 的重型版、"组合"是 A 子集），Founder 阅读时要先理清包含关系再做选择。

### 问题 4：事前验尸被外部纠错时，立场依据沉默带过

首次运行里事前验尸的 P5 流量估算 `640kbps×30min×20 次≈24GB/月` 本身算术错了（真实 ~2.8GB/月），Gemini 明确纠正。但 `[立场依据]` 字段没显式标注"事前验尸 P5 被证伪、原排序置信度降级"，只是自然地采纳 Gemini 纠正。后果：Founder 可能继续用事前排序的 P1-P5 做决策权重——而这个排序的置信度已被证伪。

事前验尸作为 skill 3.2 定义的"反自我满足锚"，必须保留其完整性——任何一条被外部证伪，整个排序的权威性都应降级。

## 讨论与决定

Founder 确认三点：核心主张用 ≤100 字可多句（放宽），其他三点采纳原改进建议。最终落地到 0.5.1：

### 1. 核心主张字段长度约束（放宽 + 量化）

- **旧**：一句话结论；禁止列事项清单
- **新**：≤100 字，可多句但围绕同一论点；禁止列事项清单或塞入战术细节。出现"另外/并且/同时还建议"等并列承接词 = 多论点信号，必须拆分
- 加到校验条件（`# 核心主张长度`）

### 2. 新增 3.3.3 Digest 与升级模板的分工规则

升级路径下 Digest 收缩为"身份证"：保留事前验尸 + 核心主张（≤100 字版本）+ 判优 + 立场 + 立场依据；`[下一步行动]` 收缩为 "见下方升级模板，Founder 勾选"。升级模板聚焦 Founder 的决策工作（事实前提 + 选项 A/B/C + 反面检验 + 决策勾选框），**不再重复**核心主张/判优/立场字段。

### 3. 升级模板加入选项互斥性约束

- 选项总数 ≤4 项（A/B/C/D）
- 选项两两互斥：禁止"组合 A+B 部分"单列
- 选项粒度对等：禁止"全套 A / A 的子步骤 / A 的重型版"嵌套关系
- 不够用就分流到"再议"下一轮 /think

### 4. 新增"事前验尸被证伪时立场依据必须降级"条款

任一路模型指出事前验尸包含**事实错误**（算术/数据/命名/时效）时，`[立场依据]` 必须显式标注"事前验尸 P<x> 被纠正，原排序置信度降级"。

## 为什么不选其他方案

### X1：一句话 ≤40 字硬约束

**否决原因**：Founder 直接判为过严。40 字装不下一个严肃的架构核心论点，强制后 Claude Code 要么牺牲精确度、要么作弊拆成多句并列。≤100 字是"可多句但必须围绕一个论点"的平衡点。

### X2：升级模式下 Digest 完全省略核心主张字段

**否决原因**：Digest 的事前验尸锚 + 立场依据锚依赖有核心主张对照。完全省略会让 Digest 失去独立可读性——Founder 如果只看 Digest 就能理解全貌，比翻升级模板快。收缩≠删除。

### X3：不管选项数量，让 Claude Code 自行决定

**否决原因**：首次运行已经证明 Claude Code 在"把所有可能方案都列出来"的诱惑下会列 6 个。硬上限（≤4）+ 互斥性约束才能机械拦住。

### X4：P 级排序降级做成"软提醒"而非校验条件

**否决原因**：Founder 的决策依赖 AI 对置信度的显式标注。软提醒等于没提醒——下次 Claude Code 还是会沉默采纳纠正。硬约束写进校验条件才有效。
