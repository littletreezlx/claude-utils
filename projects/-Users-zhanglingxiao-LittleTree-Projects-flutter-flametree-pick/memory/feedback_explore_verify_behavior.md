---
name: 探索前必须确认实际行为而非文档描述
description: AI Explore 不能用文档中的预期行为（如 ResultDialog）作为验证标准，必须先确认当前实现
type: feedback
---

AI Explore 中将文档描述（PRODUCT_BEHAVIOR.md 提到的 RandomResultDialog）当作验证标准，花了大量时间追查"弹窗不出现"的 bug——实际上当前实现根本没有弹窗，骰子动画高亮卡片就是结果。

**Why:** 文档可能描述了设计意图/未来计划，与当前实现不一致。AI 盲信文档导致误报了一个不存在的 bug，浪费了整轮探索的大部分时间。

**How to apply:**
1. 探索前如果不确定某个功能的具体行为，**先用 API 触发一次 + 截图**观察实际效果，再决定验证标准
2. 文档说"应该有 X"但 UI 没有 → 不要假设是 bug，先问用户确认
3. 当连续 5+ 次验证同一个"bug"都得到一致结果时，考虑"这可能就是正常行为"
