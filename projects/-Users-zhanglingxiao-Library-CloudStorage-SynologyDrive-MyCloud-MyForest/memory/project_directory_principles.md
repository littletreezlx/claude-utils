目录架构原则已落库到 CLAUDE.md 第 4 章。

核心规则：
- **深度**：3层骨架 + 4层容忍（≥5篇文档才建目录）+ 5层熔断（改用 Markdown 标题）
- **编号**：L1-L2 强制编号，L3 及以下不编号
- **异物隔离**：外部代码/SDK 禁止放入 Vault，移至 `SynologyDrive-MyCloud/work_sync/`

**Why:** 856 个文档、336 个目录中，4层目录最多（102个），部分达到 5-6 层。深层编号增加维护成本，深层目录降低侧边栏导航效率。

**How to apply:** 新建目录时检查深度和文档数阈值。后续执行重构时按此原则展平深层目录、去除 L3+ 编号。
