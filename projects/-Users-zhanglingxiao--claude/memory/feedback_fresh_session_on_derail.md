---
name: 跑偏时新开会话
description: 方案跑偏时应回滚代码并新开会话，避免错误上下文污染后续判断
type: feedback
---

方案连续修补仍不理想时，优先 git 回滚 + 新开会话，而非在当前会话中继续硬推。

**Why:** 当前会话中的错误中间产物（失败的尝试、错误的假设）会干扰 LLM 后续判断，导致越改越偏。新会话 = 干净上下文 = 更好的方案。来源：宝玉解读 Claude Code 团队技巧 #6 "推倒重来"。

**How to apply:** 当连续 2-3 次修补仍不满意时，建议用户回滚代码并新开会话重新描述需求，而非在污染的上下文中继续尝试。
