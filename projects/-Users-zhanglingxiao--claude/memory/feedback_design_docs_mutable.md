设计文档（`PRODUCT_SOUL.md`、`UI_SHOWCASE.md`、`design-philosophy.md` 等）描述的是**当前状态**，不是不可变的理想态。评审 UI 时应将其作为「参考背景」而非「合规标尺」。

**Why:** 用户使用 Stitch/v0 等 AI 设计工具的目的就是突破现有设计限制。如果拿现有文档当圣经去校验新设计，等于在用旧框架扼杀创新。排版、色调系统、卡片样式、导航结构都可以完全重新设计。

**How to apply:**
- 所有 UI skill（ui-vision-check、ui-vision-advance、ui-vs）加载设计文档时，态度应该是「了解现状」而非「找偏差」
- 当新设计明显优于现有设计系统时，应建议**更新文档**以跟随更好的方向，而非要求设计回退
- 评审的终极标尺是 FlameTree 四根设计支柱（Typography First / Warm Ceramic / Less but Better / Tactile Precision），不是文档中的具体色值/布局/组件规范
- 文档是活的，设计语言是演进的——文档跟着最佳实践走，不是反过来
