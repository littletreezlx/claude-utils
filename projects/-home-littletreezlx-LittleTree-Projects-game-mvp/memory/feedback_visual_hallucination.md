## 铁律：视觉描述必须严格基于实际输入

**错误做法**：描述 UI 截图时，混用预训练知识或设计文档内容替代实际观察。例如看到 CampScene 就写"战士(蓝)、弓兵(绿)、法师(紫)"，但截图里只有 1 张蓝色战士卡。

**正确做法**：
- 先说"实际看到的是：XXX"
- 用文档/代码做**解释框架**，不做**事实替代**
- 数清楚截图里实际有几个元素、什么颜色、什么布局，再输出描述

**为什么重要**：AI 会用预训练知识自动补全画面，这种幻觉在视觉相关任务中极其危险。

**如何应用**：所有 Read 截图后的描述、godot-explore / ai-qa-stories 的视觉反馈、UI 审美评估，必须以截图实物为准。文档知识只能用于解释"为什么这样设计"，不能用于描述"截图里有什么"。
