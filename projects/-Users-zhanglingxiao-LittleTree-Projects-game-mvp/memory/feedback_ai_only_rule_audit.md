**规则**：AI-Only 项目（Founder 不写代码、不做编辑器操作）里，当 AI 用某条规则反对一个操作时，先问自己"这条规则的隐含前提里有没有'人类会做 X'？AI-Only 模式下这个前提还成立吗？"。如果不成立，规则该修订而不是机械执行。

**Why**：2026-04-26 在 game-mvp 评估 godot-mcp 接入时，AI 用项目 CLAUDE.md 里"AI 不做布局微调，留给人在编辑器中拖拽"反对 `add_node` / `create_scene` 工具。Founder 反问"那我又不去做拖拽，你这个能力到底有没有？" —— 直接戳穿规则与协作模式不一致：项目早期可能预期 Founder 偶尔动手，后来全局 CLAUDE.md 声明 AI-Only 但项目级规则没同步刷新，导致 UI / 数值流程长期停滞（AI 不做、Founder 也不做）。

**How to apply**：
- 当用某条"AI 不做 X，留给 Founder/人类做 X"的规则反对操作时，强制自检三步：
  1. 这条规则写下时是否预期人类会动手？
  2. 当前协作模式（看全局 CLAUDE.md 是否声明 AI-Only）下，人类还会动手吗？
  3. 如果不会，AI 的能力 + 项目自有的反馈闭环（截图 / Debug Server / 测试）能否补上？能补就修订规则
- 高发场景：UI 编辑器操作、可视化数值调优、设计工具内拖拽、IDE 内重构等"传统上是人类用图形界面做"的任务
- 不要等 Founder 反问才发现 —— 主动审计 CLAUDE.md 中所有"AI 不做 X"条款，验证前提是否过时
- 修订规则时必须配套：闭环路径（怎么验证）+ 验证凭证（截图 / 日志 / 数值快照）+ 让位项（确实没法闭环的少数情况，明确列出）
