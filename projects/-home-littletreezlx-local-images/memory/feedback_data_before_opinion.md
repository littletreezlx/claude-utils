对不确定的技术问题（如"LLM 能否自主管理姿势多样性"），必须先跑实验拿数据，再下结论。AI 和用户的推理都可能有主观偏差。

**Why:** 2026-03-26 姿势多样性讨论中，Claude 和 Gemini 都基于推理给出了判断（Gemini 认为 LLM 缺乏卡牌计数能力，需要 Runtime Deck 方案），但用户要求先测再说。实验数据直接推翻了 Gemini 的核心判断——Euryale 70B 完全有能力自主去重。

**How to apply:**
- 遇到"X 能不能做到 Y"类问题 → 写测试脚本验证，不要凭推理下结论
- 实验脚本模板：`scripts/test_pose_freedom.py`（多组对比 + 统计分析）
- 特别是涉及 LLM 能力边界、图像模型标签理解力等问题，推理不可靠
- 对 Gemini Think 的回复也要保持独立判断，它的分析同样可能被推翻
