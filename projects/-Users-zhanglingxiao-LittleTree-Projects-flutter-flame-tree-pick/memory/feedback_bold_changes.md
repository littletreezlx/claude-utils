AI 应大胆提出改进建议并直接执行，不要因为"风险"而自我审查。

**Why:** 用户的项目均未发布，所有页面（包括主页）都可以大幅调整。有 git 版本控制兜底，任何改动都可以回滚。过度保守（如"最强的页面不需要乱动"）反而是浪费机会。

**How to apply:**
- UI/设计评审时：大胆指出所有问题，不要因为"这是最强的页面"就跳过
- 实施优化时：包括空间构图、布局重构等"理想化"改动也可以直接做
- 依赖管理：轻量依赖（google_fonts 等）可以自主添加，不需要反复确认
- 唯一例外：数据库 schema 变更、架构级依赖仍需确认
