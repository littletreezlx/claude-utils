# think skill — 进阶规则

本文件收纳 think skill 的**按需加载**内容:Dual 模式、多轮追问、升级模板细则、`--quick` 备用通道。默认 Solo 路径不需要读这里。

**何时读本文档**:
- 使用 `--dual` 调用 think.mjs 时
- 进行多轮追问(Turn 2+)时
- `[我的立场]` 命中升级路由(见主 SKILL.md 3.3.1)时
- 用户显式要求 `--quick` 时

---

## 1. Dual 模式

### 1.1 何时真正需要 Dual

Dual 默认 2x token 成本,只在以下场景有正回报:

- **真策略分歧**:两个主流立场在专业圈内都被严肃对待,Claude Code 无法独立判断哪边更对
- **影响极广**:决策会影响多个项目/多个月,多视角压 bias 价值高
- **用户显式请求**:"让两个都看看"、"Dual 跑一下"

反例(不应 Dual):
- 纯信息查询(根本不该 call think)
- Turn 2+ 追问(首轮 Dual 已建立两方立场,追问沿一方深入用 `--solo` 即可)
- Claude Code 已经有较强倾向,call think 只是想找共识(走 Solo,通过事前验尸强制挑战自己)

### 1.2 调用命令

```bash
export NVM_DIR="$HOME/.nvm" && [ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh" && node ~/LittleTree_Projects/other/nodejs_test/projects/ai/think.mjs --dual --out /tmp/think-$(date +%Y%m%d-%H%M%S).md "<prompt>"
```

`--out` 是必选(详见主 SKILL.md Step 2)。stdout 只打一行 `[think] ✓ written to <path>`,Claude Code 用 Read 读取文件拿到完整双块原文。

输出两个分隔块:

```
═══ 🧠 Gemini (google/gemini-3.1-pro-preview) — 9.4s ═══
<Gemini 原文>

═══ 🤖 GPT-5.4 (openai/gpt-5.4) — 2.8s ═══
<GPT 原文>
```

某一路失败时显示 `❌ ERROR` 和错误原因,另一路结果照常返回(`Promise.allSettled` 保证)。单路失败不算整体失败,在 Digest 里标注"X 路失败,本轮仅基于 Y 方主张"继续完成决策。两路同时失败才报错退出。

### 1.3 Dual 模式 Digest 7 字段模板

```markdown
### 🧠 Think Digest

- **[我的事前验尸]**: <Claude Code 在 call think **之前**写的事前验尸答案,原文回显(含可证伪前提)>
- **[Gemini 核心主张]**: <Gemini 核心结论,≤100 字,可多句但围绕同一论点;禁止列事项清单或塞入战术细节>
- **[GPT-5.4 核心主张]**: <GPT-5.4 核心结论,同样 ≤100 字,同样一个中心论点>
- **[模型判优]**: Gemini 胜 | GPT 胜 | 互补并采 | 两败俱伤 —— <≤60 字说明判据;若赢家与我原始倾向一致,必须补"反向情况下我会选谁、为什么这次不选"的 counterfactual>
- **[我的立场]**: 采信 | 反驳 | 升级
- **[立场依据]**: <1-2 句话,基于上述判优后,我为什么采信 / 反驳 / 升级>
- **[下一步行动]**: <具体的第一步指令,或 "仅记录,无动作"。引用编号(如 R1/BUG-2)必须紧跟一句话注释该编号指代>
```

### 1.4 模型判优(Claude Code 担任 judge)

#### 判优维度(不是全维度都给分,挑本题最相关的 1-2 维)

| 维度 | 看点 |
|------|------|
| **推理深度** | 是否追问到第一性原理,或停留在表层 |
| **盲区识别** | 是否点出我事前验尸没想到的失败模式 |
| **具体性** | 是否给出可执行判据,还是空洞的大词堆砌 |
| **反直觉检验** | 是否主动寻找反面论据,而非附和 |
| **多模态理解**(带图时) | 是否真的看懂图,还是在对文字提示发挥 |
| **时效性** | 是否引用过时/编造的事实 |

#### 反偏好校验(必跑)

Claude Code 同时是提问者+裁判,存在 confirmation bias 风险。判优结果确定后,**必须**自问:

> 赢家是否与我 call think 前的原始倾向一致?

**若一致**:在 `[模型判优]` 字段末尾追加一句 counterfactual——"反向情况下我会选谁、为什么这次不选"。理由必须具体:点出输家哪条具体论据更弱、或赢家哪条具体论据不可替代,不能只说"另一方说得没这方好"这种模糊比较。

**若校验做不下去**(说不出具体弱点):判优结果可疑,重新权衡——可能是偏好投射到了判优维度上。

**若不一致**:无需补 counterfactual,但仍需在 `[模型判优]` 说明判据。

#### "互补并采"的滥用警戒

"互补并采"**不是偷懒选项**——只有两路**指向不同维度**且**同时有价值**时才用(例:"Gemini 指出 UX 风险,GPT-5.4 指出架构债")。两路只是措辞不同、观点相同时,必须挑一方"胜"。

### 1.5 Dual 模式的 Digest 自包含规则

与 Solo 相同(见主 SKILL.md 3.2):引用外部编号必须同字段内注明指代。

## 2. 多轮追问

两路模型(Dual)或单路模型(Solo)**都无记忆**,多轮对话必须在 Prompt 里重建完整上下文。

### 2.1 滑动窗口原则

- **最近 2 轮**:保留完整原文(问题 + 模型回复 + Claude Code 判断)
- **更早轮次**(第 1 ~ N-2 轮):压缩为 bullet points "已锁定共识"

### 2.2 Prompt 格式

```
## Context
{与首轮相同 — 每轮重传,不可省略}

## Constraints
{与首轮相同}

## 对话历史

### 已锁定共识 (Locked Consensus)
{早期轮次压缩为 bullet points:}
- 共识 1: ...(如 dual 模式下两方都同意)
- 共识 2: ...(单方主张被另一方采纳)
- 放弃方案: ... (原因: ...)

### 第 N-1 轮 (完整保留)
#### 问题
{完整问题}
#### 🧠 Gemini 回复
{完整回复}
#### 🤖 GPT-5.4 回复
{完整回复 — solo 模式下删除这个分块}
#### Claude Code 判优与反馈
{Claude Code 本轮判优结果 + 分歧/补充}

### 第 N 轮 (完整保留)
{同上}

## 本轮问题
{本轮的具体追问/分歧/新需求}
```

### 2.3 多轮传递原则

1. Context 和 Constraints 每轮必须重传
2. 滑动窗口:最近 2 轮保留完整,更早轮次压缩为已锁定共识
3. 已锁定共识三要素:已达成的共识、已放弃的方案及原因、未解决的遗留问题
4. Claude Code 判优与反馈必须保留——让两方模型理解"Claude 上轮判谁赢、为什么" + "为什么要追问"。Dual 下这段天然起到**让输家看到评语**的效果,逼迫两方在下一轮都提高质量

### 2.4 Dual 多轮:两方共用同一份 Prompt

Dual 模式下,第 N 轮 Prompt **同时发给 Gemini 和 GPT-5.4**。Prompt 里既包含 Gemini 上轮回复也包含 GPT-5.4 上轮回复,这样两方都能看到对方观点并互相挑战,dual 的价值在多轮中会放大。

Solo 多轮只需保留单路历史,删除对方分块。

### 2.5 压缩风险

"已锁定共识"由 Claude Code 自己整理,存在把不同意的观点压缩成"已放弃"的 bias 风险。压缩时:
- 尽量用外部模型的原文措辞,不加自己的解读
- "已放弃方案"必须写明"被谁、用什么理由驳回",不要只写"已放弃"
- 多轮 ≥4 轮后若原始讨论被压得太狠,考虑另起一个会话重新 Context 化,而不是继续压缩

## 3. 升级模板细则

升级模板骨架见主 SKILL.md 3.3.3。本节展开关键约束的完整规则。

### 3.1 Dual 模式触发升级时的 Digest 收缩(避免双重展开)

**一旦 `[我的立场] = 升级` 且当前是 Dual 模式,Digest 必须收缩为"身份证"模式**,由升级模板承担主体展示。否则 Founder 会读到两次同样的概括——核心主张、判优、立场都被重复一遍。

| 字段 | 升级模式下的处理 |
|------|-----------------|
| `[我的事前验尸]` | **保留原文回显**(这是 skill 反仪式性锚,不可省) |
| `[Gemini 核心主张]` | **保留 ≤100 字主张**;具体战术细节下沉到升级模板的"选项"段 |
| `[GPT-5.4 核心主张]` | 同上 |
| `[模型判优]` | **保留**判优 + 判据 + 反偏好校验(身份证核心) |
| `[我的立场]` | 填"升级" |
| `[立场依据]` | **保留**,说明触发 3.3.1 哪条 |
| `[下一步行动]` | 收缩为一句话:"见下方升级模板,Founder 勾选",具体选项不在此展开 |

升级模板里**不要再重复** Gemini 核心主张 / GPT 核心主张 / 模型判优字段——Digest 已经有了。升级模板聚焦于 Founder 的决策工作:事实前提 + 选项 A/B/C + 反面检验 + 决策勾选框。

**Solo 模式触发升级时**:Digest 同样收缩(核心主张字段保留),但无"两次展开"问题,规则较 Dual 宽松。

### 3.2 选项互斥性约束

- **选项总数 ≤4 项**(A/B/C/D),禁止超过
- **两两互斥**:选 A 不能自动包含 B 的子集;"组合 A+B 部分"不单列为独立选项,若确实需要折中,把折中组合作为新的独立选项 C(要素明确列出)
- **粒度对等**:不允许"全套方案 A / A 的一个子步骤 / A 的重型版"这种嵌套关系——Founder 读时会陷入包含关系判断
- 若四个选项仍不够,用"再议"分流到下一轮 /think,不要继续膨胀选项数

### 3.3 `[Claude Code 综合推荐]` 字段强制要求

- **必须明确指向一个具体选项**(A/B/C/D 之一),不允许"看情况"、"看 Founder 偏好"、"视实测结果"等回避措辞——外部模型判断 + 事前验尸都沉淀好了,Claude Code 就是 synthesizer,拒绝推荐 = 工作没做完
- **必须包含反面对比**:不只说"选 X 的理由",还要说"为什么不是最接近的替代 Y"——这是对 Founder 决策权的尊重,让他看到权衡而不是单向推销
- **不允许"组合推荐"**("推荐 A 但也做一点 B"):互斥性约束已保证选项不重叠,推荐就是挑一个
- **长度 ≤100 字**:与核心主张字段对齐;过长 = 没想清楚,停下来压缩

### 3.4 事实前提字段的模式差异

| 会话类型 | "事实前提"字段 |
|---------|--------------|
| **交互模式** | 可省略或精简(Founder 已在现场) |
| **自主模式**(落库 to-discuss.md) | **必填**——Founder 下次看到时已脱离当时语境,需要事实重建 |

## 4. `--quick` (DeepSeek) 备用通道

### 4.1 何时用

**只在用户显式要求时使用**:"用 deepseek 想想"、"quick think"、"便宜跑一下"。

Claude Code 不得自主选 quick——Claude 没有校准过 DeepSeek 在方法论话题上与 Gemini/GPT-5.4 的质量差距,保守默认 Solo gemini/gpt。

### 4.2 限制

- **不支持图片**:组合 `--image` 会直接报错。图片讨论必须走 Gemini
- **无判优步骤**:单路回复,Digest 用 Solo 5 字段模板
- **事前验尸、反仪式性自检、升级路由**等其他约束与 Solo 保持一致,不因为"quick"而松懈

### 4.3 调用

```bash
node think.mjs --quick --out /tmp/think-$(date +%Y%m%d-%H%M%S).md "<prompt>"
```

`--out` 同样必选(详见主 SKILL.md Step 2)。Prompt 格式与 Solo 一致(主 SKILL.md "首轮 Prompt 格式")。

## 5. ADVANCED.md 与 SKILL.md 的分工纪律

- **SKILL.md**:默认 Solo 路径完整可运行。不加载 ADVANCED.md 也能完成 Solo 首轮调用 + Digest 输出 + 升级路由判断
- **ADVANCED.md**:Dual / 多轮 / 升级模板细则 / quick 通道,按需加载
- **禁止在 ADVANCED.md 定义 Solo 路径核心机制**(事前验尸、反仪式性自检、Digest 5 字段、升级路由命中条件)——这些是必读,必须留在主 SKILL.md
- 若未来发现 ADVANCED.md 的某条规则在主 Solo 路径也需要,**上移到 SKILL.md**,不要在 ADVANCED.md 让主路径强制加载
