# Role: UI Cleaner
# Task: Emergency UI Cleanup for macOS Main Page
# Context: The previous "background icon" implementation failed (looked like a glitch). The selection state is overlapping text.

## 1. REMOVE The "Ghost" Background Icons
- **Action**: Completely remove the background `Icon` layer inside the Cards.
- **Result**: The Card content should be a simple `Center(child: Text(...))`. No stacks blending icons behind text.

## 2. Refine the Card Visuals (Minimalist Style)
Create a clean distinction between selected and unselected states using **Background Color** instead of thick borders.

**Visual Logic:**
- **Container Decoration**:
  - **Shadow**: Keep `AppShadows.cardDefault`.
  - **Color**: 
    - *Is Selected*: `Color(0xFFFFF7F0)` (Very light warm orange tint).
    - *Not Selected*: `Colors.white`.
  - **Border**:
    - *Is Selected*: `Border.all(color: AppColors.primary.withOpacity(0.3), width: 1.5)` (Subtle, not harsh).
    - *Not Selected*: `Border.all(color: Colors.transparent)`.

**Typography Logic:**
- **Text Style**:
  - *Is Selected*: `color: AppColors.primary`, `fontSize: 32.sp`, `fontWeight: FontWeight.w700`.
  - *Not Selected*: `color: AppColors.textPrimary`, `fontSize: 32.sp`, `fontWeight: FontWeight.w600`.

## 3. Fix the "Overlapping Bug" (Selection Indicator)
If you need to show a checkmark or indicator for the selected item:
- **Strict Layout**: Use a `Stack`.
- **Text**: `Center` alignment.
- **Indicator**: `Positioned(top: 12.h, right: 12.w, child: Icon(Icons.check_circle, color: AppColors.primary, size: 24.sp))`.
- **Crucial**: The indicator must stay in the corner and NEVER overlap the center text.

## 4. Layout Constraints (Keep Wide)
- Maintain the `childAspectRatio: 1.3` (Landscape) for Desktop.
- Maintain `maxWidth: 1100.w`.
- Maintain `crossAxisSpacing: 24.w`.

## Summary
"Less is more." Remove the background icons. Fix the selection overlay bug by using absolute positioning for the checkmark. Use a soft orange background tint for the active state.
